## Platform

A self-hosted production environment running across infrastructure, data, services, surfaces, and the agent layer. Every line below is in production, signed, and observable — not a wishlist.

### Compute & Networking
Bare-metal Kubernetes with HA control plane and quorum witness. Zero-trust networking via Cilium and eBPF — every workload is identified by SPIFFE before its packets reach the mesh. L3/L4/L7 policies as code, mutual authentication enforced across selected namespaces. Flux for GitOps reconciliation; drift detected and re-applied automatically. Proxmox underneath for bare-metal VM management.

### Data
Cloud-native PostgreSQL (CNPG) with high availability, point-in-time recovery, and tested restore. Row-level security enforces multi-tenant isolation at the database — application code cannot bypass it by mistake. pgvector for embeddings and semantic search. S3-compatible object storage with replication. CSI block storage with synchronous replicas.

### Delivery & Supply Chain
Container images built on **zero-CVE Wolfi (Chainguard)** base. Signed with cosign; SBOM attestations generated by Syft. Kyverno admission enforces signature verification and SBOM presence at the cluster boundary. **SLSA-3 reproducible builds with deferred attestation** — the local pre-push gate produces the same artifact production receives, so production trusts the local gate.

### Security & Identity
HashiCorp Vault HA for secrets; External Secrets Operator synchronizes into Kubernetes without manifest leakage. SPIFFE workload identity issued by SPIRE. Cilium mesh-auth enforces mTLS on selected namespaces. Tetragon and Falco for runtime visibility via eBPF. Trivy for vulnerability and SBOM scanning. Pod Security Standards plus policy-as-code admission control.

### Application Layer (BFF)
A Rust backend-for-frontend on **axum + tower** that consolidates platform concerns at one boundary:

- **Tenant isolation** at request entry — header → middleware layer → request extension → typed extractor — propagated to row-level security at the database boundary.
- **Role-based access control** with declarative role-binding configuration; routes scoped per-handler.
- **Sandboxed execution** for untrusted compute via Landlock + seccomp; CPU, memory, and time bounded.
- **Token-cost governance** at the LLM boundary — budgets, hot-path evaluation, hard-stop enforcement, incident cascade.
- **Audit logging** of LLM invocations, RBAC decisions, and tenant-scoped reads/writes.
- **Multi-vendor LLM routing** with explicit failover chains, observability, and SSE streaming.

### AI & Agents
Production agent workflows with structured orchestration. **Inter-agent protocols (MCP, ACP, A2A) as first-class citizens** — agents are workload-identified, sandbox-bounded, audit-logged participants, not glorified prompts. Multi-vendor LLM across Anthropic, OpenAI, Google, and local Ollama with explicit failover chains. MCP-native tooling for structured integrations: search, code retrieval, vector lookup, browser automation. Headless browser fleet for AI-driven QA and verification.

### Surfaces
The same engineering bar applied at the user-facing edge.

- **Web** — SvelteKit + Svelte 5 and React 19 for operator dashboards and product surfaces; i18n, accessibility, dark/light/high-contrast theming, and GDPR-compliant defaults baked in.
- **Mobile** — Rust-native platform work with cosign-paired OTA updates and hardware-bound identity. The handset is treated as a first-class platform target, not a wrapper.
- **Cross-runtime crates** — many libraries are dual-mode by design: the same Rust code compiles into a CLI binary, a Kubernetes service, and a mobile runtime via feature flags. One implementation, three deploy surfaces.

### Developer Platform
The toolchain that produces all of the above.

- **Resource-aware build queue** — leases prevent OOM, no starvation between human and agent operators on the same host.
- **Worktree-per-task isolation** — every parallel work-stream gets its own filesystem view; no shared mutable state across operators.
- **Structured task tracking with checkpoints** — operator state survives session boundaries, enabling clean human-to-agent and agent-to-agent handoff.
- **AI-assisted IDE surface** — skills, hooks, and sub-agents wired into the editor; agents are first-class collaborators, not autocomplete.
- **Persistent operator memory** — learnings, project state, and feedback are typed artifacts that survive sessions and shape future behavior.

### Testing & Quality
Quality enforced before code reaches review.

- **Property-based testing** (proptest) for invariants that examples can't cover.
- **Type-driven schema sharing** — Rust types generate the TypeScript types (ts-rs) and the OpenAPI spec (utoipa); contract drift breaks the build, not production.
- **Headless browser QA** with visual regression, accessibility audits, and console-error gating.
- **Static analysis suite** — clippy, ast-grep, cargo-machete, kube-linter, license auditing — wired into pre-push hooks.
- **Trust-inverted CI** — heavy gates skip when a locally-signed pre-push attestation is present and verified.

### Observability & Operations
OpenTelemetry across services — metrics, traces, structured logs in a self-hosted stack. **eBPF-level visibility into runtime behavior** — what the kernel sees, the operator sees. Automated drift detection with remediation for recurring patterns. Tested incident playbooks with explicit rollback paths and signed runbook artifacts.

- **Build, don't consume** — Rust-native implementations over npm-glue and SaaS lock-in.
- **Self-hosted by default, sovereign by design** — bare-metal Kubernetes in Germany, GDPR-compliant, no foreign cloud.
- **Production-grade from day one** — observability, mutual TLS, signed builds, and SBOM attestations are not afterthoughts.
